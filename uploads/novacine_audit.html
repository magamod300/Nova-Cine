🔴 CRÍTICO #1 R18 completamente roto en modo web (HTML puro)
⚠️
Línea 972 de index.html: if (!IS_ELECTRON) { showState(...); return; } — en navegador el usuario ve error inmediatamente, sin ningún fallback.

El scraping de xvideos se hace desde Node.js (main process) para evitar CORS y antivirus. Correcto en Electron, pero en web puro no hay alternativa implementada.

Fix para modo web:

// eporner tiene CORS abierto — funciona en browser sin proxy
const url = `https://www.eporner.com/api/v2/video/search/?query=${q}&per_page=24&format=json`;
const res = await fetch(url);            // ✅ sin CORS
const data = await res.json();
// data.videos[] → cada video tiene .embed (URL iframe directa)
🔴 CRÍTICO #2 Scrapers "Buscar Latino" fallan en web por dependencia de proxies
⚡
Los scrapers (Cuevana3, CineCalidad, AnimeFLV, ok.ru) usan proxyFetch() que en modo web pasa por allorigins.win o corsproxy.io. Ambos bloquean frecuentemente estas URLs piratas y tienen alta latencia.

En Electron esto no es un problema porque nativeFetch() usa Node.js directamente y evita CORS. El issue es solo para el modo web.

El problema real: missav, jable.tv y sitios JAV no tienen API pública ni CORS abierto. El scraping de listas solo funciona desde el proceso Node de Electron. En modo web, la única opción viable es eporner (CORS abierto) + iframes directos.

Estrategia por capa:

eporner API — funciona en web Y Electron
CORS abierto. Endpoint: https://www.eporner.com/api/v2/video/search/?query={q}&per_page=24&format=json. Cada video tiene campo .embed con iframe URL directa. Reemplaza o complementa xvideos.
xvideos (actual) — solo Electron
Correcto mantenerlo como fuente primaria en Electron. El IPC search-r18 en Node evita CORS y AV. Problema: detección de miniaturas por regex frágil — considerar mejorar el parser.
missav.ws / jable.tv — solo Electron (Node proxy)
Para JAV: buscar en https://missav.ws/search/{código} desde Node, extraer URLs de embed. No implementable en web puro por CORS total. Requiere fetch desde main process y parsear HTML.
vidsrc.to / embed.su con IDs adultos TMDB
Ya anotado en NOTAS.md. Estos servidores indexan algunos títulos adultos de TMDB. Añadir como servidores del tab R18 cuando el tipo es r18.
/* eporner API — sin CORS, sin proxy, sin Node */
async function searchEporner(q) {
  const url = `https://www.eporner.com/api/v2/video/search/?query=${encodeURIComponent(q)}&per_page=24&format=json&thumbsize=medium&order=top-rated`;
  const res = await fetch(url);
  const data = await res.json();
  return (data.videos || []).map(v => ({
    id:       v.id,
    title:    v.title,
    thumb:    v.thumbs?.[0]?.src || null,
    embedUrl: v.embed,      // ← URL iframe directa ✅
    duration: v.length_min,
    views:    v.views,
    isR18direct: true,
  }));
}
El problema de Doraemon / Beautiful in Black — cómo resolverlo

Este es el problema más complejo. El sistema actual busca en Cuevana/Pelisplus con el título en inglés de TMDB. Muchos títulos están indexados con nombre en español distinto.

1
Buscar con título en español de TMDB (fix inmediato)
TMDB tiene el campo title en español si pides language=es-MX. Antes de scraping, hacer un GET al detalle del film en español y usar ese título para la búsqueda en Cuevana/Pelisplus.
// Obtener título en español antes de buscar:
const r = await fetch(`https://api.themoviedb.org/3/movie/${S.tmdbId}?api_key=${KEY}&language=es-MX`);
const d = await r.json();
const titleES = d.title || S.title;  // usar en scrapers
2
Panel admin (Ctrl+Shift+A) para mapeo manual — ya planeado
Para casos como Doraemon donde el título no coincide con ningún scraper automático: tabla editable en localStorage con tmdbId → URL custom. Antes de cualquier búsqueda automática, consultar este mapa. Es la solución definitiva para el problema.
3
Ampliar "Buscar Latino" con más sitios (DivxTotal, Gnula, Repelis)
DivxTotal y Gnula tienen catálogos de español latino más completos para series latinoamericanas y doblajes específicos. En Electron funcionan via Node. En web → proxy (poco fiable).
// DivxTotal — desde Node, español latino
nodeFetchRaw(`https://www.divxtotal.run/?s=${encodeURIComponent(title)}`)
// Gnula — buen catálogo LA
nodeFetchRaw(`https://gnula.nu/?s=${encodeURIComponent(title)}`)
4
Torrents en español → DivxTotal, PeliculasYSeries, MejorTorrent
Ya documentado en NOTAS.md. Son la mejor fuente para doblajes completos en LAT. El problema es que requieren scraping de HTML desde Node. Para series con pocos torrents en YTS/EZTV (inglés), estos sitios tienen la versión doblada que falta.

Lo que está bien implementado ✅
IS_ELECTRON detection
El flag IS_ELECTRON y el patrón nativeFetch() degradan gracefully. Código bien estructurado para dual-mode.
Ad-block en 3 capas
Main process + session playerSess + did-attach-webview. Bloquea 60+ dominios. Sólido y completo.
WebTorrent con Range HTTP
Servidor local con soporte Range correcto para seeking en <video>. Content-Type forzado a video/mp4 funciona bien para MKV también.
SubDL sin API key
Buena decisión reemplazar OpenSubtitles. SubDL funciona bien para películas en español.
CORS proxy doble (allorigins + corsproxy.io)
El fallback entre dos proxies es inteligente. Solo falla cuando ambos bloquean el dominio objetivo.
Trailers via Invidious
Evita el Error 153 de YouTube embed. Lista de 4 instancias con fallback automático. Funciona en web y Electron.
ZIP extractor nativo
Parser ZIP sin dependencias para los .srt de SubDL. Uso de zlib.inflateRawSync. Elegante.
reqId para race conditions
El contador S.reqId previene resultados de búsquedas anteriores que llegan tarde. Detalle importante.
