A — Provider ES-first para CinePro (Cuevana3 / Pelisplus)

1 día TypeScript Cómo escribir el provider
CinePro usa el framework @omss/framework. Cada provider es una clase TypeScript en src/providers/{nombre}/{nombre}.ts que extiende BaseProvider. Solo necesitas implementar un método async fetch(tmdbId, type) que retorna Source[].
// src/providers/pelisplus/pelisplus.ts
import { BaseProvider, Source } from '@omss/framework';
export class PelisplusProvider extends BaseProvider {
  name = 'Pelisplus';
  async fetch(tmdbId: number, type: 'movie'|'tv', opts?): Promise<Source[]> {
    // 1. Obtener título ES de TMDB
    // 2. Buscar en pelisplus.lat/?s={titulo}
    // 3. Scrapear el embed URL de la página del resultado
    // 4. Resolver el embed a .m3u8 (opcional, puede ser iframe)
    return [{ url, type:'hls', language:'es', quality:'1080p', provider:'Pelisplus' }];
  }
}
Pelisplus.lat es WordPress con endpoints /wp-json/wp/v2/posts?meta_key=tmdb_id&meta_value={id} — más fácil que scraping HTML. Cuevana3.lat tiene /api/movies/{tmdbId} no documentado pero funcional. El paso más difícil es resolver el embed a .m3u8 (filemoon, streamtape, streamwish) pero CinePro ya tiene extractores para algunos de esos.

C — Torrentio API para LAT (ya parcialmente implementado en v4.11)

Listo para usar API pública sin key
Torrentio tiene API REST pública. No requiere cuenta ni key.
// Endpoint:
https://torrentio.strem.fun/stream/movie/{imdbId}.json
https://torrentio.strem.fun/stream/series/{imdbId}:{season}:{episode}.json

// Response: { streams: [{ title: "Dune...[LAT][1080p] Wolfmax4k", url, ... }] }
// Filtrar español:
const esp = streams.filter(s => /LAT|ESP|DUAL|Spanish|Latino/i.test(s.title));
Grupos que etiquetan en español: Wolfmax4k, MejorTorrent, Cinecalidad, HDARG, Telly. Los streams son magnets (gratis, lento) o links HTTP de Real-Debrid (~3€/mes, instantáneo). Lo que falta implementar: mostrar esos streams como botones reproducibles en el player — los links HTTP de Debrid van directo a playMp4().

E — Discord Rich Presence (Electron)

1-2 horas npm discord-rpc — sigue funcionando en 2025
// package.json: "discord-rpc": "^4.0.1"
// main.js:
const DiscordRPC = require('discord-rpc');
const rpc = new DiscordRPC.Client({ transport: 'ipc' });
const clientId = 'TU_APP_ID'; // crear en discord.com/developers

rpc.login({ clientId }).catch(console.warn);

// IPC desde renderer:
ipcMain.handle('set-activity', (_e, data) => {
  rpc.setActivity({
    details: `Viendo ${data.title}`,
    state: data.isSeries ? `T${data.s}·E${data.e}` : data.year || '',
    startTimestamp: Date.now(),
    largeImageKey: 'novacine_logo',
    largeImageText: 'NovaCine',
    buttons: [{ label: 'Ver en NovaCine', url: `https://novacine.workers.dev/#movie/${data.id}` }]
  }).catch(()=>{});
});
Pasos: (1) Crear app en discord.com/developers → copiar Application ID. (2) En Assets subir el logo de NovaCine como "novacine_logo". (3) npm install discord-rpc. (4) Llamar ipcRenderer.invoke('set-activity', {...}) desde el renderer al hacer click en un servidor. Funciona aunque el user no tenga Discord abierto (falla silenciosamente).

F — Backup automático jsonbin.io (sin código extra de servidor)

Activo 2026 jsonbin.io — requiere registro gratuito
// Crear cuenta en jsonbin.io → X-Master-Key
// Primera vez (crear bin):
const r = await fetch('https://api.jsonbin.io/v3/b', {
  method:'POST',
  headers:{'Content-Type':'application/json','X-Master-Key':KEY,'X-Bin-Name':'NovaCine'},
  body: JSON.stringify(exportData())
});
const { metadata } = await r.json();
localStorage.setItem('nc_jbin_id', metadata.id);

// Auto-save cada 30 min:
setInterval(async () => {
  const id = localStorage.getItem('nc_jbin_id');
  if (!id) return;
  await fetch(`https://api.jsonbin.io/v3/b/${id}`, {
    method:'PUT',
    headers:{'Content-Type':'application/json','X-Master-Key':KEY},
    body: JSON.stringify(exportData())
  });
}, 30 * 60 * 1000);
Alternativa sin cuenta: kvdb.io — genera un bucket ID aleatorio sin registro, gratis, perfect para esto. https://kvdb.io/TU_BUCKET_ID/novacine-backup.

G — Colecciones TMDB (Sagas) — ya en la API, 0 código extra

Gratis — ya disponible El response de /movie/{id} ya incluye la colección
Cuando llamas a GET /movie/{tmdbId}, el response ya incluye belongs_to_collection: {id, name, poster_path} si la película pertenece a una saga. Solo hay que leerlo.
// En openContent, después de tmdbFetch:
if (m.belongs_to_collection) {
  const col = await tmdbFetch(
    `https://api.themoviedb.org/3/collection/${m.belongs_to_collection.id}?api_key=${KEY}&language=es-MX`
  );
  // col.parts = [{id, title, poster_path, release_date}, ...] — todos los filmes
  // Mostrar como scroll row "Saga: John Wick (4 filmes)"
}
Esto ya está en la caché TMDB (1h) así que no cuesta requests adicionales si ya abriste la película. Una de las features de más impacto con menos trabajo.

H — Buscador por persona (actor/director)

2-3 horas TMDB /search/person + /person/{id}/movie_credits
// Buscar persona:
GET /search/person?query=Keanu+Reeves&language=es-MX
// Filmografía:
GET /person/{id}/combined_credits?language=es-MX
// Response: { cast: [{id, title, poster_path, release_date, media_type}], crew: [...] }
Implementación en NovaCine: cuando el autocomplete no encuentra películas pero sí personas (el endpoint /search/multi ya devuelve personas), mostrarlas con icono diferente en el dropdown. Click en persona → cargar filmografía como grid. /search/multi ya está disponible en TMDB y devuelve media_type: "person".

I — PiP nativo + Speed control (5-10 líneas cada uno)

Trivial Solo aplica a fuentes CinePro (cineVideo)
// PiP — añadir a los botones del player:
function togglePip(){
  const v=$('cineVideo');
  if(!v||v.style.display==='none') return toast('Solo con fuentes CinePro','err');
  if(document.pictureInPictureElement) document.exitPictureInPicture();
  else v.requestPictureInPicture().catch(e=>toast('PiP no soportado','err'));
}

// Speed control — select flotante encima del video:
<select onchange="$('cineVideo').playbackRate=+this.value" style="display:none" id="speedSel">
  [0.5, 0.75, 1, 1.25, 1.5, 2].map(s=>`<option value="${s}"${s===1?' selected':''}>${s}x</option>`)
</select>
// Mostrar speedSel solo cuando cineVideo está activo
PiP en Chrome/Edge/Safari moderno. Firefox necesita user gesture explícito. El speed control es puro HTML nativo — video.playbackRate es estándar desde HTML5.
